[Aurelia](https://aurelia.io/home) is a JavaScript framework for web, mobile and desktop that leverages simple conventions to empower your creativity.

## Step 1: Set Up Your Aurelia Project

Set up an Aurelia app with [`npx`](https://www.npmjs.com/package/npx) and move into the project directory, choosing the option **Default ESNext App**:

<Snippet dark text="npx aurelia-cli new && cd aurelia-app" />
<Caption>Initializing an Aurelia app with <Link href="https://www.npmjs.com/package/npx">npx</Link> and moving into the project directory.</Caption>

In the `./aurelia_project/aurelia.json` file, under the `platform` key, change the output directory for the project to `public`:

```diff
"platform": {
    "hmr": false,
    "open": false,
    "port": 8080,
-   "output": "dist"
+   "output": "public"
}
```

<Caption>
  Changing the output directory in the{' '}
  <InlineCode>/aurelia_project/aurelia.json</InlineCode> file.
</Caption>

## Step 2: Deploying Your Aurelia App with ZEIT Now

With an Aurelia app set up, it is ready to deploy live with [ZEIT Now](/docs/v2).

Using [Now CLI](/download), deploy your app with **a single command**:

<Snippet dark text="now" />
<Caption>Deploying your Aurelia app with Now CLI.</Caption>

If you want to deploy your Aurelia app when you push to a Git repository, you can use either [ZEIT Now for GitHub](/docs/v2/git-integrations/zeit-now-for-github), [ZEIT Now for GitLab](/docs/v2/git-integrations/zeit-now-for-gitlab), or [ZEIT Now for Bitbucket](/docs/v2/git-integrations/zeit-now-for-bitbucket) to have your project automatically deployed on every push, and the [production domain](https://zeit.co/docs/v2/custom-domains/#adding-a-domain) updated on push to master.
